# 二、项目背景

## 需求背景

国内异宠饲养人群规模持续扩大，2025年，饲养异宠的人群持续上升，其中饲养水族的宠主上升3.9个百分点，占比为17.3%；饲养爬行类的宠主上升2.4个百分点，占比为13.5%；饲养鸟类的宠主上升0.9个百分点，占比为8.7%；饲养啮齿类的宠主上升0.8个百分点，占比为7.6%。小宠异宠市场规模111.06亿元，同比增长10%+，增速持续领跑宠物行业大盘，目前购买用户人数已达2000万+</cite>。

哺乳异宠（含雪貂、仓鼠、龙猫、蜜袋鼯等）是异宠中互动性最强、与人情感连接最紧密的一类。小型哺乳类宠物市场发展火热，与爬宠相比，哺乳类异宠能和人类产生更多互动</cite>，这也是本产品选择聚焦该品类的核心原因——用户对"养得好不好""出问题该怎么办"的咨询需求远高于纯观赏型异宠。

但这一群体长期面临结构性供给缺口，集中体现在三方面：

1. **看病难、看病贵是普遍痛点。** 《2025年中国宠物行业白皮书（消费报告）》显示，30%以上的异宠消费者遭遇"看病难、看病贵"，20%因喂养知识不足导致宠物死亡。宠主饲养异宠过程中，遇到的主要问题是给异宠看病的医院较少、异宠商品质量参差不齐、买到不健康的异宠、给异宠看病贵，占比均在30%以上；其次是遇到喂养问题不知如何解决、不了解异宠的行为，占比分别在20%以上。
    
2. **异宠兽医资源极度稀缺，是结构性而非偶发问题。**即便是上海这样的超一线城市，能够给异宠看病的宠物医院也不过四五家，异宠市场虽大，但配套的医药条件还远远满足不了需求。不少饲养者在被问及"异宠生病了怎么办"时坦言"上网搜方法，治不好就只能自生自灭"；稀缺的异宠医院、高达数千元的治疗费用，共同构成了"售后真空"。一线城市专门开设异宠门诊的医院，单次就诊客单价大约在800—1000元，进一步推高了用户"先上网问一问"的真实需求。
    
3. **雪貂的高发疾病具有"隐匿性强、易被忽视"的特点，凸显日常问答介入的价值。** 雪貂作为哺乳异宠中互动性最高的品种，存在三类高发内分泌 / 肿瘤疾病：内分泌肿瘤发病率高达53%，其中以肾上腺皮质肿瘤和胰岛素瘤为主，淋巴瘤次于前两者。这些疾病早期症状极易被新手误判为正常的行为或外观变化——例如肾上腺疾病的早期信号包括拉尿时徘徊几次才拉出来、鼠尾、身上某部位掉毛或变少干涩、脂溢性皮炎、精神亢奋爱咬人咬东西打架等；淋巴瘤则通常发生在2到5岁之间，中年雪貂可能无症状多年，或仅表现为非特异性的、时好时坏的迹象，常见表现为食欲减退、虚弱、嗜睡和体重减轻。这意味着用户描述"我家貂最近老掉毛""精神有点亢奋"这类看似日常的问题时，AI 若只会做表面的养护建议，会错失早期预警的机会——这正是本产品设计"行为/外观异常类"中间判断层的核心动因。
    

> 口径说明：以上数据分别引自派读《2025年中国宠物行业白皮书（消费报告）》、《2026年中国宠物行业白皮书（消费报告）》及宠业家与淘天宠物联合发布的《2025中国小宠异宠行业发展白皮书》，引用时均标注来源年份，不做跨口径混用。

## 产品范围与定位

- **核心定位**：聚焦哺乳异宠（雪貂为主品种，仓鼠 / 龙猫 / 蜜袋鼯等为次要覆盖品种）的 AI 问答工具，不做社区、不做养护记录工具、不做认证体系。
    
- **MVP 形态**：Web 应用。
    
- **核心功能边界**：AI 问答（开放式对话）+ 资源地图（高危场景配套出口）。资源地图不算范围蔓延，而是高危识别后安全闭环的必要组成部分。
    
- **商业模式**：一期完全免费；二期商业化方向 `TBD`，本版不展开。
    
- **外部依赖**：高德地图开放平台 API（Web 端 JS API + 地点检索），用于高危场景下的附近异宠兽医 / 急诊查询。
    
- **知识来源**：通用大模型能力 + 轻量化 RAG 知识库增强（雪貂为主品种的养护 / 常见病知识深度最高），RAG 语料具体规模和收录范围由产品团队后续决定，本版 PRD 仅界定架构方向。

|异常处理|—|图片上传格式 / 体积不符时明确提示；AI 服务超时 / 失败时提供重试机制；网络中断时缓存用户当前输入。|

## 4.2 AI 问答 Agent 设计

### 三层判断逻辑

|   |   |   |
|---|---|---|
|层级|触发条件|回答策略|
|日常养护类|喂食、环境、笼具、行为训练等常规养护问题|开放回答，无需安全话术限制，可给出具体、可执行的建议|
|行为 / 外观异常类|用户以日常口吻描述行为或外观变化（掉毛、性格变化、排泄异常的频率/姿态等），但未直接使用"生病""症状"等词汇|AI 需结合品种特异性高发疾病知识，判断该描述是否可能指向潜在健康风险；若有风险，需在回答中提示用户关注并观察，建议必要时记录变化、择期就医，但不直接归为"高危"|
|症状 / 疾病类|用户描述明确的疾病症状（如呕吐、出血、抽搐、无法进食、难产等）|切换至安全话术：仅给出可能性参考（不确诊）、紧急程度提示、可执行的家庭养护建议、是否需要就医的指引；全程标注"AI 辅助参考，不替代执业兽医诊断"|

### 轻量化 RAG 架构（方向性设计）

- **定位**：作为 LLM 原生回答能力的检索增强补充，非完全依赖知识库回答。
    
- **语料范围**：以雪貂的养护知识、常见病（含肾上腺疾病、胰岛素瘤、淋巴瘤等高发病）为知识深度最高的核心语料；其余哺乳异宠品种（仓鼠 / 龙猫 / 蜜袋鼯等）语料深度可适当降低。
    
- **语料规模与具体收录范围**：`TBD`，由产品团队结合开发资源后续决定，本 PRD 仅明确架构方向。
    
- **元数据建议**：每条语料建议标注来源、品种、是否健康相关，便于后续校验和迭代（具体实现方式留待开发阶段细化）。

# 判断路径
1. 问题类型判断：日常养护类 / 行为外观异常类 / 症状疾病类，三选一。
2. 若为日常养护类：直接给出具体、可执行的建议，语气自然，不附加免责声明。
3. 若为行为外观异常类：结合品种特异性高发疾病知识判断潜在风险；
   有风险时提示用户关注并建议观察记录，语气不过度引起恐慌，但要清晰传达"值得留意"。
4. 若为症状疾病类：
   a. 列出可能性（按可能性排序，简述判断依据），不下确诊结论。
   b. 给出紧急程度提示（可观察 / 建议就医 / 立即就医）。
   c. 给出当前可执行的家庭养护建议。
   d. 若判定为"立即就医"（高危），在回复中用加粗/独立段落醒目标注
      "请立即就医"，并提示可使用资源地图查找附近异宠兽医/急诊。
   e. 全程标注"本回答为 AI 辅助参考，不替代执业兽医诊断"。

# Constraints（安全红线）
- 严禁确诊：不得输出"确诊为 XX 病"式结论，只能给可能性。
- 严禁处方：不得给出具体处方药及剂量。
- 宁可误报，不可漏报：对疑似高危情况一律从严判断，倾向于提示就医。
- 数据诚实：RAG 未覆盖或信息不足时，明确说明信息有限，不编造品种特异性数据，
  建议线下就医确认。
- 次要品种（雪貂以外的哺乳异宠）若知识覆盖不足，需明确告知用户"该品种相关
  信息暂时有限"，避免用雪貂知识套用到其他品种身上。

# 六、评测需求

采用"安全层 + 体验层"双评测框架。

## 安全层（底线指标，不可妥协）

- **高危信号识别准确率**：评测集由产品团队与异宠兽医专家协作构建，覆盖雪貂高发急症 / 重症的真实场景样本（如严重脱水、难产、持续出血、无法进食、肾上腺疾病晚期症状等），定义标准答案后工程化为可执行评测用例。
    
- **上线前必须经异兽医专家抽检评审**，确认 AI 在真正高危场景下的识别能力达标后方可上线。
    
- 具体准确率阈值与漏报率上限：`TBD`，待评测集构建时与兽医专家共同确定。
    
- 模型或 Prompt 迭代后必须回归测试。
    

## 体验层（产品质量指标）

- **问答满意度**：每次问答后用户点赞 / 点踩的比例。
    
- **回答采纳率**：用户是否认为该回答解决了问题（可通过后续行为间接衡量，如是否继续追问同一问题）。
    
- **复访 / 复问频率**：用户是否在非首日再次发起提问，衡量产品是否真正被当作"日常工具"使用。
    

> 安全层与体验层并非互相替代关系：安全层保底线，体验层定优劣。两者均需在上线后持续监控。

# 七、兜底策略

|   |   |   |
|---|---|---|
|触发条件|系统行为|用户感知|
|RAG 检索无相关素材 / 置信度过低|AI 明确说明信息有限，不臆造；健康相关一律建议线下就医|提示"该情况信息有限，建议尽快咨询能看异宠的兽医"并引导资源地图|
|AI 服务超时 / 失败|重试机制（指数退避）|提示"分析超时，请稍后重试"并提供重新提问按钮|
|AI 疑似高危但不确定|从严判定为"建议立即就医"，醒目标注但不强制拦截|回复中加粗 / 独立段落呈现就医提示，附资源地图入口|
|高德地图无结果 / 定位失败|降级为手动输入城市 / 关键词检索|提示"未找到附近点位，可手动搜索"|
|图像上传格式 / 过大|前端拦截并提示|"当前格式 / 体积不支持，请上传符合要求的图片"|
|网络中断|前端缓存用户当前输入，恢复后可继续|"网络已断开，请检查后重试"|
|非雪貂品种知识覆盖不足|AI 明确告知该品种相关信息有限，避免套用雪貂知识作答|提示"该品种相关信息暂时有限，建议咨询专业异宠兽医"|
