# Skills 目录说明

本目录存放三个「小型哺乳异宠」场景 Skill 的**原文归档**，以及后端如何使用它们的约定。
它们取代了原 PRD 里的 RAG 方案——用结构化、人工维护的场景知识模块增强 AI 输出，不依赖向量库。

## 三个 Skill 与三层判断的对应关系

| Skill 原文 | 对应三层判断 | 职责 | 输出结构 |
|---|---|---|---|
| `小型哺乳异宠饲养顾问 Skill.md` | 日常养护类 (DAILY_CARE) | 饮食/笼舍/温湿度/清洁/新宠到家/多宠 | 温和判断→物种要点→可做什么→避坑→总结 |
| `小型哺乳异宠行为分析 Skill.md` | 行为/外观异常类 (BEHAVIOR_ANOMALY) | 咬人/躲藏/破坏/乱拉/攻击/情绪 | 接情绪→去人格化→原因→今天做1-2步→红灯转医疗 |
| `小型哺乳异宠医疗边界与急症分级 Skill.md` | 症状/疾病类 (SYMPTOM_DISEASE) + 高危 | 症状四档分级+现场止损+就医沟通 | 接情绪→确认红灯→问关键→现场动作→电话模板 |

## 关键架构约定

1. **按意图路由**：AI 先判断问题类型，只注入**对应那一个** Skill 的浓缩 prompt 片段（不是全塞三个）。
   - 浓缩片段在代码里维护（`web/src/lib/ai/skills/`），本目录原文是"事实来源"，片段是其提炼。

2. **医疗红线强制叠加（双保险）**：
   - 关键词预检：命中医疗 Skill 的红线关键词（呕吐/出血/抽搐/不吃/误食/呼吸异常/中暑…）→ 强制走医疗 Skill。
   - 模型判断：prompt 内也要求模型识别健康信号。
   - 两道保险，宁可误报不可漏报（符合 PRD 安全红线）。

3. **四档分级 → urgency 字段**：
   - 绿色(日常维护) / 黄色(短时观察) → `observe`
   - 橙色(当天联系医生) → `see_vet`
   - 红色(急诊思维) → `emergency` → 触发"请立即就医" + 资源地图

4. **物种化**：当前宠物档案的品种字段注入 Skill，让建议物种化，不跨物种套用。

5. **Skill 覆盖物种**：仓鼠/龙猫/蜜袋鼯/雪貂/臭鼬/豚鼠/花枝鼠/刺猬/兔类及其他小型哺乳异宠。

## 迭代方式

- 改 Skill 效果 → 改 `web/src/lib/ai/skills/` 下对应片段；原文文档作为参考基准。
- Prompt 与业务逻辑解耦，符合 PRD 可维护性要求。
