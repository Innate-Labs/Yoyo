## 二、推荐的技术细节（我来定，无异议即执行）

### 架构
- **框架**：Next.js 14+ App Router，前后端同仓
- **数据库**：PostgreSQL；ORM 用 Prisma
- **RAG 向量存储**：`pgvector` 扩展（复用 PostgreSQL，不额外引入向量库），embedding 用国产 embedding API（如通义 text-embedding）
- **鉴权**：手机号 + 验证码；服务端签发 JWT / session cookie
- **状态**：会话/消息流式返回（SSE）实现打字机效果

### 数据模型（初版）
- `users`：id、手机号、创建时间
- `pets`：id、user_id、品种(species)、昵称、年龄、性别（**一对多**，支持多只）
- `conversations`：id、user_id、pet_id（当前会话绑定的宠物）、标题、创建/更新时间
- `messages`：id、conversation_id、角色(user/assistant)、文本、图片URL、question_type（AI判定的三层标签，内部存储不展示）、created_at
- `feedbacks`：id、message_id、user_id、点赞/点踩
- `events`：埋点事件表（event_name、user_id、payload JSON、created_at）
- `rag_chunks`：id、内容、embedding(vector)、来源、品种、是否健康相关（元数据）

### AI 问答 Agent 落地
- 单次请求内完成 PRD 4.2 的三层判断（日常养护 / 行为外观异常 / 症状疾病）
- Prompt 与业务逻辑解耦（独立文件维护），可迭代
- 结构化输出：模型返回 `{ question_type, is_high_risk, answer }`，便于埋点与高危提示渲染
- **高危提示**：`is_high_risk=true` 时，回答中加粗/独立段落呈现"请立即就医"，并在消息末尾渲染"资源地图"入口按钮，不做强制弹窗
- **图片安全**：多模态看图同样走安全话术；涉及患处/排泄物图片一律"可能性参考+就医引导"，不做影像确诊
- **免责声明**：仅症状疾病类回答标注"AI 辅助参考，不替代执业兽医诊断"；日常/行为异常类不加

### 图片上传
- 格式：jpg/png/webp；体积：≤ 10MB（前端拦截+后端校验）
- 存储：本地 MVP 存文件系统 / 后续换对象存储；数据库存 URL

### 兜底（按 PRD 第七章实现）
- RAG 无命中/置信度低 → 明确说明信息有限 + 引导就医
- AI 超时/失败 → 指数退避重试 + "分析超时，请重试"
- 高德无结果/定位失败 → 降级手动输入城市/关键词
- 网络中断 → 前端缓存当前输入
- 非雪貂品种覆盖不足 → 明确告知信息有限，不套用雪貂知识

## 三、可上线 MVP 需要用户提供的东西（阻塞项）

1. **短信服务 Key**：阿里云/腾讯云短信（SignName + TemplateCode + AccessKey）
2. **高德开放平台 Key**：Web 服务 API（地点检索）+ Web端 JS API（地图展示）
3. **大模型 Key**：阿里云百炼 DashScope API Key（Qwen-VL-Max）+ embedding 权限
4. **备案/域名/隐私政策**：面向国内用户上线需要，MVP 可先本地跑通，上线前补齐

> 在拿到 Key 之前，我会用可切换的 mock 实现让主流程先跑通，拿到 Key 填 `.env` 即切真实。

## 五、我对 PRD 的补充建议（供参考）

- **隐私政策/用户协议**：收集手机号+宠物信息+位置，上线前建议补齐（法务需求）
- **内容审核**：用户上传图片建议加基础审核（违规图拦截），MVP 可先简单处理
- **会话管理**：历史对话建议支持删除/重命名（PRD 未提，体验相关）
- **图片高危**：PRD 安全话术未单独覆盖"图片高危识别"，已在上面按同等安全边界落地
